# Database Table Outline - Hybrid Relational Approach

## Philosophy
- **Indexed columns** for filtering, reporting, and data integrity
- **JSON columns** for variable/optional data
- **Polymorphic relations** for flexible media handling

---

| # | Table | Purpose |

## Part 2: Appointment System Plan

### Option A: Centralized Appointments Table (Recommended)

Create one `appointments` table that links to any service record:

| Column | Type | Notes |
|--------|------|-------|
| id | BigInteger (PK) | |
| appointment_date | Date | |
| appointment_time | Time | |
| service_type | Enum | 'vaccination', 'kapon', 'adoption_interview' |
| service_id | BigInteger | FK to medical_records or adoption_applications |
| status | String | scheduled/completed/cancelled/no_show |
| notes | Text | nullable |
| created_at | Timestamp | |
| updated_at | Timestamp | |

**How it connects:**

```
appointments
    ├── service_type = 'vaccination' → links to medical_records (type='vaccination')
    ├── service_type = 'kapon' → links to medical_records (type='kapon')
    └── service_type = 'adoption_interview' → links to adoption_applications
```

### Option B: Separate Tables per Service

- `vaccination_appointments`
- `kapon_appointments`  
- `adoption_interview_schedules` (already in adoption_applications as zoom_date)

---

## Recommendation

**Use Option A** - Single centralized table because:

1. **Unified calendar** - Admin sees all appointments in one view
2. **Efficient queries** - Filter by date, service type, status
3. **Scalable** - Easy to add new service types
4. **Reuses existing columns** - Can use appointment_date from existing tables + add time

---

## Implementation Notes

### If using Option A:

1. **Keep existing appointment_date** in medical_records/adoption_applications as "requested date"
2. **Add to appointments table**:
   - `appointment_time` (specific time slot)
   - `service_type`, `service_id` (polymorphic-style link)
   - `status` for tracking

3. **No changes needed** to existing tables - just add appointments table

### Questions for clarification:

1. Do you need **daily slot management** (e.g., max 10 vaccinations per day)?
2. Should appointments be **specific time slots** (9:00 AM, 9:30 AM) or just dates?
3. Do you need **staff/doctor assignment** to appointments?
4. For adoption - is the interview via zoom or in-person (or both)?

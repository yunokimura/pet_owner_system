# Database Table Outline - Hybrid Relational Approach

## Philosophy
- **Indexed columns** for filtering, reporting, and data integrity
- **JSON columns** for variable/optional data
- **Polymorphic relations** for flexible media handling

---

## Final Tables (6 tables)

| # | Table | Purpose |
|---|-------|---------|
| 1 | `medical_records` | Vaccination + Kapon combined |
| 2 | `medical_record_pets` | Pivot: record → pets |
| 3 | `adoption_applications` | All adoption data |
| 4 | `adoption_selected_pets` | Pivot: application → adoption_pets |
| 5 | `media` | Polymorphic: any image/file |
| 6 | `medical_record_agreements` | Separate for kapon agreements (indexed) |

---

## 1. `medical_records` (Hybrid: indexed + JSON)

| Column | Type | Indexed | Notes |
|--------|------|---------|-------|
| id | BigInteger (PK) | Yes | Auto-increment |
| user_id | BigInteger (FK) | Yes | For user lookups |
| owner_id | BigInteger (FK) | Yes | For owner lookups |
| type | Enum | Yes | 'vaccination' or 'kapon' |
| appointment_date | Date | Yes | For scheduling reports |
| status | String | Yes | pending/approved/completed/rejected |
| metadata | JSON | No | Type-specific data (see below) |
| created_at | Timestamp | - | |
| updated_at | Timestamp | - | |

**metadata JSON structure:**
```json
// Vaccination
{
  "last_anti_rabies_date": "2025-01-15",
  "recent_surgery": "no",
  "confirmation": true
}

// Kapon
{
  "general_agreements": {
    "secure_cage": true,
    "keep_indoors": true,
    "keep_dry": true,
    "full_authority": true,
    "valid_id": true,
    "location": true
  }
}
```

## 6. `media` (Polymorphic)

| Column | Type | Indexed | Notes |
|--------|------|---------|-------|
| id | BigInteger (PK) | - | |
| model_type | String | Yes | App\Models\MedicalRecord, App\Models\Pet, etc. |
| model_id | BigInteger | Yes | ID of related model |
| file_name | String | - | Original file name |
| file_path | String | - | Storage path |
| mime_type | String | - | image/png, image/jpeg, etc. |
| file_size | Integer | - | In bytes |
| description | String | - | Optional description |
| created_at | Timestamp | - | |

**Polymorphic Usage:**
```php
// Link to medical record
$record->media()->create(['file_path' => '...']);

// Link to pet
$pet->media()->create(['file_path' => '...']);

// Link to adoption application
$application->media()->create(['file_path' => '...']);
```

**Use cases:**
- Medical record photos (kapon)
- Pet photos (registration, adoption)
- Home photos (adoption)
- Valid IDs (adoption)
- Any future media needs

## Why This Is Future-Proof

1. **Reporting**: Easy to query by status, type, date ranges, assigned officer
2. **Data Integrity**: Foreign keys and indexed columns ensure consistency
3. **Media Flexibility**: Polymorphic relation handles any future media type
4. **Scalability**: JSON for variable data, relational for core data
5. **Maintenance**: Clear separation between queryable and flexible data
